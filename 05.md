## 1. 核心目标

### 2.1 监控工具类 (`monitor.py`)
**功能**：在后台开启一个守护线程，每隔 0.5 秒采集一次系统资源。

```python
# monitor.py
import psutil
import time
import threading

class SystemMonitor:
    def __init__(self):
        self.is_running = False
        self.metrics = [] # 存储 CPU 和内存数据

    def start(self):
        """启动监控线程"""
        self.is_running = True
        # daemon=True 表示主程序退出时，该线程会自动结束
        t = threading.Thread(target=self._collect)
        t.daemon = True
        t.start()

    def _collect(self):
        """采集循环"""
        while self.is_running:
            # 【关键点】interval=None 表示立即获取当前值，不阻塞
            cpu = psutil.cpu_percent(interval=None)
            memory = psutil.virtual_memory().percent
            
            self.metrics.append({"cpu": cpu, "memory": memory})
            
            # 打印实时日志
            print(f"[Monitor] CPU: {cpu}% | Memory: {memory}%")
            
            # 控制采集频率，避免过度占用 CPU
            time.sleep(0.5)

    def stop(self):
        """停止监控并返回历史数据"""
        self.is_running = False
        return self.metrics

if __name__ == "__main__":
    # 本地调试代码
    monitor = SystemMonitor()
    monitor.start()
    time.sleep(3)
    data = monitor.stop()
    print(f"采集到 {len(data)} 条数据")
```

### 2.2 Pytest 集成钩子 (`test_framework/conftest.py`)
**功能**：利用 `fixture` 实现“测试开始自动启动监控，测试结束自动停止并生成报告”。

```python
# test_framework/conftest.py
import pytest
import sys
import os

# 引入根目录的 monitor.py
sys.path.append(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
from monitor import SystemMonitor

@pytest.fixture(scope="session", autouse=True)
def setup_monitor():
    """
    scope="session": 整个测试会话只运行一次
    autouse=True: 自动应用到所有测试，无需手动调用
    """
    monitor = SystemMonitor()
    print("\n>>> 启动性能监控守护线程...")
    monitor.start()
    
    yield # ------------------ 测试执行区域 ------------------
    
    print("\n>>> 测试结束，停止监控...")
    data = monitor.stop()
    
    # 数据处理与报告输出
    if data:
        avg_cpu = sum([d['cpu'] for d in data]) / len(data)
        avg_mem = sum([d['memory'] for d in data]) / len(data)
        print(f"\n===== 性能监控报告 =====")
        print(f"采集样本数: {len(data)}")
        print(f"平均 CPU: {avg_cpu:.2f}%")
        print(f"平均 内存: {avg_mem:.2f}%")
        print(f"=========================")
```

## 3. 常见报错与解决办法

### 问题现象：测试通过，但没有显示性能报告
**现象**：
```text
...
PASSED
>>> 测试结束，停止监控...
(后面没有性能报告了)
```

**原因分析**：
1.  **采集卡顿**：`psutil.cpu_percent(interval=1)` 默认会阻塞（等待）1 秒钟来计算 CPU 值。
2.  **速度不匹配**：本地 SQLite 数据库测试非常快（小于 0.5 秒）。
3.  **结果**：监控线程刚启动就卡在 `interval=1` 那里，还没等到数据返回，主程序里的测试已经跑完了，并调用了 `stop()`。监控线程醒来时发现 `is_running=False`，于是直接退出，导致 `metrics` 列表为空。

**解决办法**：
修改 `monitor.py` 中的 `_collect` 方法：
1.  将 `psutil.cpu_percent(interval=1)` 改为 `psutil.cpu_percent(interval=None)`，避免阻塞。
2.  在循环末尾手动添加 `time.sleep(0.5)`，控制采集频率。
