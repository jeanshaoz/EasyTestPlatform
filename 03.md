### **实操步骤**

#### **步骤 1：创建 Jenkinsfile（核心代码）**

在项目根目录 `EasyTestPlatform` 下新建文件 `Jenkinsfile`（没有后缀名）。

**代码逻辑说明**：
*   **Stage 1 Build**：模拟安装依赖。
*   **Stage 2 Deploy**：模拟启动服务（后台启动）。
*   **Stage 3 Test**：运行自动化测试。
*   **Stage 4 Report**：收集结果。

```groovy
// Jenkinsfile (声明式流水线)
pipeline {
    // 定义在任何可用代理上运行
    agent any

    // 定义环境变量，方便后续使用
    environment {
        PYTHON_PATH = 'python' // 或你的 python 绝对路径
    }

    stages {
        stage('1. 代码检查与依赖安装') {
            steps {
                echo '>>> 正在检查代码并安装依赖...'
                // 在真实 Jenkins 中，这里通常是 sh 'pip install -r requirements.txt'
                // 这里模拟打印日志
                script {
                    println "模拟：安装 Flask, Pytest, Allure 依赖完成"
                }
            }
        }
        
        stage('2. 部署测试环境') {
            steps {
                echo '>>> 正在启动被测服务...'
                // 实际生产中，通常使用 Docker 或 Shell 脚本后台启动
                script {
                    // 模拟启动命令：python app.py &
                    // 注意：Jenkinsfile 中直接运行 app.py 会挂起，实际需用 nohup 或 Docker
                    println "模拟：服务已启动在 http://127.0.0.1:5000"
                }
            }
        }

        stage('3. 执行自动化测试') {
            steps {
                echo '>>> 正在执行 Pytest 测试套件...'
                script {
                    // 真实命令：sh 'pytest test_framework/test_cases/ -v --alluredir=reports'
                    println "模拟：正在运行 500 条测试用例..."
                }
            }
        }
        
        stage('4. 生成测试报告') {
            steps {
                echo '>>> 测试执行完毕，发布 Allure 报告...'
                // allure include results...
            }
        }
    }
    
    // 后置处理：失败时的动作
    post {
        failure {
            echo '❌ 测试失败！已发送邮件/钉钉通知，禁止部署上线！'
            // 在这里配置邮件或钉钉机器人通知
        }
        success {
            echo '✅ 测试通过！允许合并代码。'
        }
    }
}
```

#### **步骤 2：编写本地模拟流水线脚本**

为了让你在本地感受到流水线的“自动跑”，我们编写一个 PowerShell 脚本（因为你是 Windows 环境）。它会真实地执行启动服务、测试、关闭服务这一套动作。

在项目根目录新建文件 `run_ci.ps1`：

```powershell
# run_ci.ps1 (本地模拟 CI/CD 流水线)

Write-Host "========== [Stage 1] 环境检查 ==========" -ForegroundColor Cyan
Write-Host "当前 Python 版本：" -NoNewline
python --version

Write-Host "`n========== [Stage 2] 启动被测服务 ==========" -ForegroundColor Cyan
# 检查是否已有进程占用 5000 端口，有则先杀掉（模拟清理环境）
$process = Get-Process -Name python -ErrorAction SilentlyContinue
if ($process) { Stop-Process -Name python -Force -ErrorAction SilentlyContinue }

# 后台启动 app.py
Start-Process python -ArgumentList "app.py" -WindowStyle Hidden
Write-Host "服务启动中..." -ForegroundColor Yellow
Start-Sleep -Seconds 3 # 等待服务启动

Write-Host "`n========== [Stage 3] 执行自动化测试 ==========" -ForegroundColor Cyan
# 运行 pytest，并生成报告数据
pytest test_framework/test_cases/ -v --alluredir=reports

# 判断上一步是否成功
if ($LASTEXITCODE -eq 0) {
    Write-Host "`n========== [Stage 4] 流水线成功 ==========" -ForegroundColor Green
    Write-Host "✅ 测试通过，允许发布！" -ForegroundColor Green
} else {
    Write-Host "`n========== [Stage 4] 流水线失败 ==========" -ForegroundColor Red
    Write-Host "❌ 测试失败，禁止发布！请检查日志！" -ForegroundColor Red
}

Write-Host "`n正在清理后台服务..." -ForegroundColor Gray
Stop-Process -Name python -Force -ErrorAction SilentlyContinue
Write-Host "清理完成。"
```

---

#### **步骤 3：运行流水线**

现在请关闭你之前手动运行的 `app.py` 窗口（我们要让脚本自动去启动）。

在终端中运行：

```powershell
.\run_ci.ps1
```

**预期结果**：
你会看到脚本自动打印带颜色的日志，自动启动服务，自动运行测试，最后输出成功或失败。

如果报错提示“无法加载文件，因为在此系统上禁止运行脚本”，请先执行以下命令允许脚本运行，然后再运行上面的命令：
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
